## 빠른 A+B
브론즈 IV

| 시간 제한 | 메모리 제한 |
|--------|--------|
| 1 초 | 512 MB |

### 문제
본격적으로 for문 문제를 풀기 전에 주의해야 할 점이 있다. 입출력 방식이 느리면 여러 줄을 입력받거나 출력할 때 시간초과가 날 수 있다는 점이다.

C++을 사용하고 있고 cin/cout을 사용하고자 한다면, cin.tie(NULL)과 sync_with_stdio(false)를 둘 다 적용해 주고, endl 대신 개행문자(\n)를 쓰자. 단, 이렇게 하면 더 이상 scanf/printf/puts/getchar/putchar 등 C의 입출력 방식을 사용하면 안 된다.

Java를 사용하고 있다면, Scanner와 System.out.println 대신 BufferedReader와 BufferedWriter를 사용할 수 있다. BufferedWriter.flush는 맨 마지막에 한 번만 하면 된다.

Python을 사용하고 있다면, input 대신 sys.stdin.readline을 사용할 수 있다. 단, 이때는 맨 끝의 개행문자까지 같이 입력받기 때문에 문자열을 저장하고 싶을 경우 .rstrip()을 추가로 해 주는 것이 좋다.

또한 입력과 출력 스트림은 별개이므로, 테스트케이스를 전부 입력받아서 저장한 뒤 전부 출력할 필요는 없다. 테스트케이스를 하나 받은 뒤 하나 출력해도 된다.

자세한 설명 및 다른 언어의 경우는 [이 글](http://www.acmicpc.net/board/view/22716)에 설명되어 있다.

[이 블로그 글](http://www.acmicpc.net/blog/view/55)에서 BOJ의 기타 여러 가지 팁을 볼 수 있다.

### 입력
첫 줄에 테스트케이스의 개수 T가 주어진다. T는 최대 1,000,000이다. 다음 T줄에는 각각 두 정수 A와 B가 주어진다. A와 B는 1 이상, 1,000 이하이다.

### 출력
각 테스트케이스마다 A+B를 한 줄에 하나씩 순서대로 출력한다.

### 예제 입력 1
> 5
> <br>1 1
> <br>12 34
> <br>5 500
> <br>40 60
> <br>1000 1000

### 예제 출력 1
> 2
> <br>46
> <br>505
> <br>100
> <br>2000

- 수학

### 제출
```java
import java.io.BufferedReader;
import java.io.BufferedWriter;
import java.io.IOException;
import java.io.InputStreamReader;
import java.io.OutputStreamWriter;
import java.util.StringTokenizer;

public class Main{
    public static void main(String[] args) throws NumberFormatException, IOException{
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(System.out));

        int cnt = Integer.parseInt(br.readLine());
        StringTokenizer st;

        for(int i = 0; i<cnt;i++)
        {
            st = new StringTokenizer(br.readLine());
            bw.write((Integer.parseInt(st.nextToken())+Integer.parseInt(st.nextToken()))+"\n");
        }
        bw.flush();
        bw.close();
    }
}
```

### 풀이
이 문제를 Scanner를 이용해서 풀어보면 제한시간을 초과하는 문제가 발생한다.

여기서의 핵심은 Scanner 대신 다른 방법을 통해 입출력을 사용해야 한다는 것이다.
왜냐하면 Scanner는 기본적으로 정규식을 통해 문자열을 파싱하는 과정이 내부적으로 포함되어있기 때문에 속도가 느리기 때문이다.

다른 방법으로 BufferdReader를 사용하는 방법이 있다. 이 클래스는 readLine()메서드만 제공하고 정규식 처리 없이 단순하게 문자열을 읽기만 하도록 내부적으로 동작하기 때문에 속도측면에서 Scanner보다 더 빠르다.

InputStreamReader를 사용하는 방법도 있지만 이번에는 BufferedReader를 사용해서 문제를 해결했다.

사용법은 다음과 같다.

```java
BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(System.out));
```

입력을 위한 BufferedReader와 출력을 위한 BufferedWriter 객체를 생성한다.

InputStreamReader는 바이트 기반의 입력 스트림을 문자 스트림으로 변환하는 클래스이고 System.in은 표준 입력 스트림으로 바이트 기반이다.
<br>System.in을 통해 바이트 데이터를 읽고 InputStreamReader를 통해 문자로 변환한 후에 이것을 BufferedReader가 받는 것이다.

OutputStreamWriter는 문자기반의 출력을 바이트 기반의 출력 스트림으로 변환하는 클래스이고 System.out은 표준 출력 스크림으로 바이트 기반이다.

그 다음으로 BufferdReader는 라인 단위로 문자열을 가져오기 때문에 분리가 필요하다면 StringTokenizer를 통해 분리하는 과정이 필요하다.

```java
StringTokenizer st; // 변수 생성

st = new StringTokenizer(br.readLine()); // 객체 생성 

Integer.parseInt(st.nextToken()); // nextToken을 나눠진 Token단위로 하나씩 가져올 수 있음.
```
이를 통해 입력에 가능하다.

출력은 write()메서드를 통해 가능하다.

```java
bw.write((Integer.parseInt(st.nextToken())+Integer.parseInt(st.nextToken()))+"\n");

bw.flush(); // 강제 버퍼 비우기
bw.close(); // BuffuredWriter 닫기
```
주의할 점은 버퍼를 사용하면 비워야 한다는 것 인데 혹시나 버퍼에 남아있는 내용이 있을때 다음 출력에서 그 내용들이 같이 출력되는 문제를 방지하기 위함이다.
