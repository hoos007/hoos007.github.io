### 문제
두 자연수 A와 B가 주어진다. 이때, A+B, A-B, A*B, A/B(몫), A%B(나머지)를 출력하는 프로그램을 작성하시오.

### 입력
두 자연수 A와 B가 주어진다. (1 ≤ A, B ≤ 10,000)

### 출력
첫째 줄에 A+B, 둘째 줄에 A-B, 셋째 줄에 A*B, 넷째 줄에 A/B, 다섯째 줄에 A%B를 출력한다.

### 예제 입력 1
> 7 3

### 예제 출력 1
> 10
> 4
> 21
> 2
> 1

### 알고리즘 분류
- 수학
- 구현
- 사칙연산

***
### 제출
```java
import java.util.Scanner;

public class Main{
    public static void main(String[] args)
    {
        Scanner stdin = new Scanner(System.in);
        int n1 = stdin.nextInt();
        int n2 = stdin.nextInt();

        System.out.println(n1 + n2);
        System.out.println(n1 - n2);
        System.out.println(n1 * n2);
        System.out.println(n1 / n2);
        System.out.print(n1 % n2);
    }
}
```

### 풀이
`java.util.Scanner`를 import하고

`System.in`의 내용을 사용하는 `Scanner`객체를 생성한 뒤

`nextInt()`함수를 통해 `int`값을 입력받을 수 있다.
```java
import java.util.Scanner; // Scanner import

Scanner stdin = new Scanner(System.in); // Scaneer 객체 생성

stdin.nextInt(); // 다음 int값 찾는 함수
```
이를 통해 입력이 가능하다.

```java
System.out.println(n1 + n2);
System.out.println(n1 - n2);
System.out.println(n1 * n2);
System.out.println(n1 / n2);
System.out.print(n1 % n2);
```
이를 통해 출력이 가능하다.

`println()`함수를 통해 출력을 할 수 있다. 이 함수는 줄 내림을 포함하고있다.

`print()`함수를 통해 출력을 할 수 있다. 이 함수는 줄 내림을 포함하지 않는다.

#### 자바 표준 입출력 클래스
사용자가 프로그램과 대화하기 위해서는 사용자와 프로그램 사이의 입출력을 담당하는 수단이 필요합니다.

자바에서는 모든 것이 객체로 표현되므로, 입출력을 담당하는 수단 또한 모두 객체입니다.

C언어의 printf() 함수나 scanf() 함수처럼 자바에서는 System이라는 표준 입출력 클래스를 정의하여 제공하고 있습니다.

이러한 System 클래스는 java.lang 패키지에 포함되어 제공됩니다.

System 클래스에는 표준 입출력을 위해 다음과 같은 클래스 변수(static variable)가 정의되어 있습니다.

1. System.in
2. System.out
3. System.err

자바에서는 System.in 스트림을 사용하여 표준 입력 작업을 수행합니다.

또한, System.out 스트림이나 System.err 스트림을 사용하여 표준 출력 작업을 수행합니다.

#### java.util 클래스 Scanner
java.lang.Object
<br>java.util.Scanner
<br>모든 구현된 인터페이스:
<br>Closeable, AutoCloseable, Iterator\<String\>

<br>public final class Scanner
<br>extends Object
<br>implements Iterator\<String\>, Closeable
<br>정규 표현식을 사용하여 기본 타입과 문자열을 구문 분석할 수 있는 간단한 텍스트 스캐너입니다.
<br>Scanner는 입력을 토큰으로 분리하는데, 기본적으로 공백에 일치하는 구분자 패턴을 사용합니다. 결과로 생성된 토큰은 다양한 next 메서드를 사용하여 다른 타입의 값으로 변환될 수 있습니다.

예를 들어, 이 코드는 사용자가 System.in에서 숫자를 읽을 수 있게 합니다:

```java
Scanner sc = new Scanner(System.in);
int i = sc.nextInt();
```

또 다른 예로, 이 코드는 파일 myNumbers의 항목에서 long 타입을 할당할 수 있게 합니다:

```java
Scanner sc = new Scanner(new File("myNumbers"));
while (sc.hasNextLong()) {
    long aLong = sc.nextLong();
}
```

스캐너는 공백 이외의 구분자를 사용할 수도 있습니다. 이 예제는 문자열에서 여러 항목을 읽어옵니다:

```java
String input = "1 fish 2 fish red fish blue fish";
Scanner s = new Scanner(input).useDelimiter("\\s*fish\\s*");
System.out.println(s.nextInt());
System.out.println(s.nextInt());
System.out.println(s.next());
System.out.println(s.next());
s.close(); 
```

다음 출력을 생성합니다:

```
1
2
red
blue
```

동일한 출력은 다음 코드를 사용하여 한 번에 모든 네 개의 토큰을 구문 분석하여 생성할 수도 있습니다:

```java
String input = "1 fish 2 fish red fish blue fish";
Scanner s = new Scanner(input);
s.findInLine("(\\d+) fish (\\d+) fish (\\w+) fish (\\w+)");
MatchResult result = s.match();
for (int i=1; i<=result.groupCount(); i++)
    System.out.println(result.group(i));
s.close(); 
```

스캐너가 사용하는 기본 공백 구분자는 Character.isWhitespace에 의해 인식됩니다. reset() 메서드는 이전에 변경되었는지 여부와 관계없이 스캐너의 구분자 값을 기본 공백 구분자로 다시 설정합니다.

스캔 작업은 입력을 기다리는 동안 차단될 수 있습니다.

next()와 hasNext() 메서드 및 기본 타입 동반 메서드(예: nextInt() 및 hasNextInt())는 먼저 구분자 패턴과 일치하는 입력을 건너뛴 다음 다음 토큰을 반환하려고 시도합니다. hasNext와 next 메서드는 추가 입력을 기다리는 동안 차단될 수 있습니다. hasNext 메서드가 차단되는지 여부는 해당 next 메서드가 차단될 것인지와 연관이 없습니다.

findInLine(java.lang.String), findWithinHorizon(java.lang.String, int), 그리고 skip(java.util.regex.Pattern) 메서드는 구분자 패턴과 독립적으로 작동합니다. 이 메서드들은 입력의 구분자에 관계없이 지정된 패턴과 일치하려고 시도하며, 구분자가 관련이 없는 특별한 상황에서 사용할 수 있습니다. 이 메서드들은 더 많은 입력을 기다리는 동안 차단될 수 있습니다.

스캐너가 InputMismatchException을 던질 때, 스캐너는 예외를 일으킨 토큰을 전달하지 않아, 다른 방법을 통해 검색하거나 건너뛸 수 있습니다.

구분자 패턴의 유형에 따라 빈 토큰이 반환될 수 있습니다. 예를 들어, 패턴 "\\s+"는 구분자의 여러 인스턴스와 일치하기 때문에 빈 토큰을 반환하지 않습니다. 구분자 패턴 "\\s"는 한 번에 하나의 공백만 전달하므로 빈 토큰을 반환할 수 있습니다.

스캐너는 Readable 인터페이스를 구현하는 모든 객체에서 텍스트를 읽을 수 있습니다. 기본 readable의 Readable.read(java.nio.CharBuffer) 메서드 호출이 IOException을 던지면, 스캐너는 입력의 끝에 도달했다고 가정합니다. 기본 readable에서 발생한 가장 최근의 IOException은 ioException() 메서드를 통해 검색할 수 있습니다.

Scanner가 닫히면, 소스가 Closeable 인터페이스를 구현하는 경우 입력 소스도 닫힙니다.

Scanner는 외부 동기화 없이는 다중 스레드 사용에 안전하지 않습니다.

특별히 언급되지 않는 한, Scanner의 메서드에 null 매개변수를 전달하면 NullPointerException이 발생합니다.

스캐너는 useRadix(int) 메서드를 사용하여 다르게 설정되지 않는 한 숫자를 10진수로 해석하는 것이 기본값입니다. reset() 메서드는 이전에 변경되었던 여부와 관계없이 스캐너의 기수 값을 10으로 다시 설정합니다.

지역화된 숫자

이 클래스의 인스턴스는 표준 형식뿐만 아니라 스캐너의 로캘 형식에서 숫자를 스캔할 수 있습니다. 스캐너의 초기 로캘은 Locale.getDefault() 메서드에서 반환되는 값
