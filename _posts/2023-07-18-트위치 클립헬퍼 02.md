### 1. python도 패키지가 있나?
우선 본격적인 방법 조사에 들어가기 이전에 파이썬은 자바 패키지처럼 정리할만한 기능이 있는지 궁금했습니다. 그래서 gpt 한테 물어봤습니다.(gpt-4)

> 질문: 파이썬에도 자바의 패키지 같은게 있니?

>답변: 네, 파이썬에도 자바의 패키지와 유사한 기능을 하는 모듈과 패키지가 있습니다. 
>
>파이썬에서는 서로 연관된 코드들을 묶어 하나의 모듈로 만들 수 있습니다. 그리고 이런 모듈들을 묶은 것을 패키지라고 합니다. 패키지를 사용하면 코드를 더 잘 조직화하고 관리할 수 있으며, 코드의 재사용성을 높일 수 있습니다.
>
>파이썬에서 패키지를 만드는 방법은 다음과 같습니다:
>
>1. 패키지로 사용할 디렉토리를 만듭니다.
>2. 해당 디렉토리 내에 `__init__.py` 파일을 생성합니다. 이 파일은 해당 디렉토리가 패키지의 일부임을 파이썬에게 알려주는 역할을 합니다.
>3. 필요한 모듈을 해당 디렉토리에 추가합니다.
>
>이렇게 생성한 패키지는 `import` 문을 사용해 다른 파이썬 스크립트에서 사용할 수 있습니다. 예를 들어, `mypackage`라는 이름의 패키지가 있고, 그 안에 `mymodule`라는 모듈이 있다면, 다음과 같이 사용할 수 있습니다:
>
>```python
>import mypackage.mymodule
>```
>
>또한 파이썬에서는 `pip`라는 패키지 관리 시스템을 통해 다른 사람들이 만든 패키지를 쉽게 설치하고 사용할 수 있습니다.

#### Get Started
##### 1. Client ID, Client Secret 받기
`Get Started` 탭을 보니까 먼저 어플리케이션을 등록하라고 나오네요
그래서 등록했습니다.
등록하니 `Client ID`랑 `Client Secret`을 얻을 수 있었어요.
메모장에 기록했습니다.

##### 2. token 받기
그 다음에는 토큰을 받아오라고 하네요
받아오는 방법은 이렇습니다.

요청방법
```bash
curl -X POST 'https://id.twitch.tv/oauth2/token' \
-H 'Content-Type: application/x-www-form-urlencoded' \
-d 'client_id=<your client id goes here>&client_secret=<your client secret goes here>&grant_type=client_credentials'
```
응답 형식
```json
{
  "access_token": "jostpf5q0puzmxmkba9iyug38kjtg",
  "expires_in": 5011271,
  "token_type": "bearer"
}
```
보니까 저 주소로 `post`형식으로 보내고
헤더에는  `Content-Type` 지정해주고
데이터로 body에 `client_id`랑 `client_secret`, `grant_type`이렇게 세개 보내주라고 나와있네요. `client_id`랑 `client_secret`은 내가 가지고 있는거 보내주고 `grant_type`은 정해진 값(client_credentials)을 보내주면 되는것 같습니다.
body에는 json형식으로 해서 보내줘야겠습니다.

응답으로는 토큰이랑 만료일 토큰 타입이 오네요
근데 저 `expires_in`안에있는 저 숫자가 뭔지 모르겠어요
그래서 gpt한테 물어보기로 했습니다.
>질문:
{
  "access_token": "jostpf5q0puzmxmkba9iyug38kjtg",
  "expires_in": 5011271,
  "token_type": "bearer"
}
내가 토큰을 요청하면 이렇게 보내준다는데 expires_in 값이 뭘 말하는걸까?

>대답:
>expires_in은 토큰의 유효 기간을 초 단위로 나타내는 값입니다.
>
>예를 들어, expires_in이 5011271이라면, 토큰이 발급된 시점으로부터 약 5011271초 후에 토큰이 만료되는 것을 의미합니다.
>
>이렇게 제공되는 토큰의 유효 기간 정보는 보안을 유지하기 위해 중요한 요소입니다. 토큰이 만료된 후에는 다시 인증을 받아 새로운 토큰을 받아야 합니다. 이는 잠재적인 불법적인 사용을 방지하는 데 도움이 됩니다.

초단위라고 하네요

내친김에 `token_type`도 뭔지 물어봤습니다
>질문:
token_type은 뭐야?

>대답:
>`token_type` 필드는 OAuth 2.0 인증 토큰 응답에서 제공되며, 반환된 액세스 토큰의 유형을 명시합니다. 이는 클라이언트가 토큰을 어떻게 사용해야 하는지를 나타내는 지표입니다.
>
>예를 들어, "bearer"라는 `token_type` 값은 Bearer 토큰을 나타냅니다. Bearer 토큰은 토큰 자체를 인증 수단으로 사용하는 토큰입니다. 즉, 해당 토큰을 가진 사람이나 클라이언트가 요청을 보낼 수 있는 권한이 있다는 것을 의미합니다. Bearer 토큰은 일반적으로 헤더에 포함되어 전송되며, 'Authorization: Bearer <토큰 값>' 형식을 따릅니다.
>
>다른 유형의 토큰으로는 MAC(Messaged Authentication Code) 토큰이 있습니다. 이 토큰은 클라이언트가 서버로 요청을 보낼 때 마다 서명을 생성하는데 사용됩니다.
>
>각 토큰 유형은 다른 보안 특성과 요구 사항을 가질 수 있으므로, `token_type` 필드는 클라이언트가 알맞게 토큰을 사용하도록 안내하는 중요한 정보입니다.

읽어보니 토큰타입에 따라 그 토큰의 목적과 권한 역할이 다르다 정도로 이해하면 될것같습니다.

##### 3. 사용자 정보 가져오기
마지막 부분에 사용자 정보를 가져오는 방법이 나와 있었습니다.
사실 이 다음에 나오는 채널 정보 가져오는 법을 먼저 봤는데 거기서 숫자로된 아이디를 주는 상황이 나옵니다.
숫자 아이디 이게 뭔가 해서 뒤져봤는데 여기서 주네요.

게다가 여기서 프로필 이미지랑
오프라인 상태 화면을 가져올수 있네요?
유용하게 사용할 수 있을것 같습니다.

요청 방법
```bash
curl -X GET 'https://api.twitch.tv/helix/users?login=twitchdev' \
-H 'Authorization: Bearer jostpf5q0puzmxmkba9iyug38kjtg' \
-H 'Client-Id: wbmytr93xzw8zbg0p1izqyzzc5mbiz'
```
응답 형식
```json
{
  "data": [
    {
      "broadcaster_type": "partner",
      "created_at": "2021-07-30T20:32:28Z",
      "description": "Supporting third-party developers building Twitch integrations from chatbots to game integrations.",
      "display_name": "TwitchDev",
      "id": "141981764",
      "login": "twitchdev",
      "offline_image_url": "https://static-cdn.jtvnw.net/jtv_user_pictures/3f13ab61-ec78-4fe6-8481-8682cb3b0ac2-channel_offline_image-1920x1080.png",
      "profile_image_url": "https://static-cdn.jtvnw.net/jtv_user_pictures/8a6381c7-d0c0-4576-b179-38bd5ce1d6af-profile_image-300x300.png",
      "type": "",
      "view_count": 6652509
    }
  ]
}
```
Get 요청으로 `login`파라메타를 같이 보내주면 됩니다.
로그인 파라메타에는 유저의 `로그인 아이디`가 들어갑니다.
헤더에는 형식에 맞춰 `토큰`과 `Client_id`를 보내줍니다.
그러면 숫자 아이디를 반환해줍니다.


#### 채널 정보 가져오기

요청 방법
```bash

-H 'Client-Id: wbmytr93xzw8zbg0p1izqyzzc5mbiz'
```

>url: https://api.twitch.tv/helix/channels
>요청방법: GET
>요청 파라메타:
>1. broadcaster_id
>String
>필수
